```shell
我的LLM工具目前大多都是 CopilotPro，

1. 最近几个月一直投入开发专注于某些开源项目中长期被用户强烈需求（比如最近的Kazumi，见截图）、但因难度高或优先级低而一直无人（或很少人）完成。这些功能往往卡住了项目的进一步普及和生态发展，我的目标是通过系统性实现它们，为社区提供实质性推动。
2. 除了编程之外，我用 Nanobot 构建了一套个人写作工作流，实现“写作赋能”：通过分析用户历史笔记和文章，自动提炼出个人的写作风格（包括词汇偏好、句式习惯、语气节奏、结构偏好等），然后基于此风格进行高质量的文章生成、续写和智能校对。
```

```shell
我主要用 LLM 做 Vibe Coding，提升个人工作效率。我用多模型路由（发挥各模型优势），将日常开发效率提升3-5倍，快速从idea到可用原型/产品。具体成果有用AI重构和迭代了手上不少香米，优化代码质量，减少了手动维护时间、采用多Agent协作 + 长链推理的工作流：

1 用强推理模型（如Claude Opus/Sonnet系列或Gemini Pro）进行任务分解和整体架构设计；
2 代码生成用Grok/Cursor强项、创意/UI用GPT系列、多模态/分析用Gemini，充分发挥各模型优势；
3 多Agent并行审查（代码审查Agent + 测试Agent），结合自我反思循环（long-chain reasoning），自动处理错误、重试和优化
4 输出直接进入本地环境或部署管道，支持持续迭代
```
