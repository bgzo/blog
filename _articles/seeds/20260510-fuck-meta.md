去年 20251116 就尝试注册过 Meta 的开发者，但是死活都过不去开发者认证的接收短信这一步， Google Voice 和 gifffgaff 都收不到验证码，去年得出的结论是无解：

> ip 被 meta 送中了，meta/facebook 系的 ip 规则十分严格，甚至可以说严格到变态。而且最致命的是无解
> https://www.v2ex.com/t/1154689

今天不死心，还是想要把 Telegram 机器人 接入 Threads[^integrate-api]，我想，自己的手机号都是英国、美国的，不应该是手机号的问题，所以想着要不新注册一个账号吧。

[^integrate-api]: 两种实现方案，1. 盗版 API 2.正版 API https://github.com/tirthpatell/threads-go 还是想走正版

然后拿 Giffgaff 去网页注册 FB，果真拿到验证码了，但总是提示失败，我尝试多点了几次，成功是成功了，但要求我验证是真人，需要拍摄脸部视频、然后上传，接着等待，卧槽，原来是被封了。

没招了，然后去下载了 Whatsapp 的 APP，注册成功，比 FB 丝滑一下，INS 也能成功从 Whatsapp 跳转登陆成功，就是 FB 还是想开头那样，直接锁死不让用，一点办法都没有。

只能等待了，又浪费我两个小时的时间，甚至 FB、INS、Threads 里面都没有客服，连提交申诉的地方都没有，Whatsapp 好一点，但也全是 AI ，找不到真人。

浪费时间。

我实在不理解为什么 Meta 所有的 API（包括 Threads） 都要放在 Facebook 里面，不理解为什么我邮箱账号为什么一直收不到验证码，更不理解为什么我新注册的账号为什么只有 FB 是停用的状态。

或许这就是 Meta 吧，Fuck Meta! Fuck Facebook!
