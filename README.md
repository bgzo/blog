# 见字如面

写点什么吧，在我决定把这个博客关闭之前。

2021年末，那时候我还没有搞清楚 GitHub Action、Jekyll 和 GitHub Pages 的关系，以及他们的工作原理，我看到 [@dzhavat](https://github.com/dzhavat) 的博客，我觉得很不错，就一点点模仿，加入自己觉得不错的元素点缀，框架（TailwindCSS），再一点点打磨，最后就成了现在这个样子。

数据源拉取也是照着 [yihong0618/2022](https://github.com/yihong0618/2022) 的 GitHub Action 一点点改的，就连用Issue写博客也是听 [捕蛇者说](https://pythonhunter.org/) 某期博客时被启发到的，觉得很有意思，就想试试。

实际[最终结果](https://github.com/bGZo/blog/tree/2021/10/issue-blog)还不错，你可以理解我成功把 GitHub Issue 做成了一个网站来对外展示，内容来源于 Issue，评论区也来源于 ISSUE。

可能是做的时候没多想，做完后很长一段时间我才意识到，自己并不喜欢在 ISSUE 里面写博客，除了从 Logseq/Obsidian 搬运过来太麻烦之外，还有污染自己的GitHub 正常工作，常年关不掉的 Issue 对我这种强迫症来说是一种变相折磨。还有一些别的原因我在[一篇文章](https://note.bgzo.cc/weekly/1269-archive-blog-written-by-github-issue)里面也总结过，不赘述了。

总之，[这些年兜兜转转，想法千奇百怪，走过不少弯路](./../changelogs.html)，不过那些都想流水一般流走了，我想我也应该像我使用这些工具一样，是流动的，拥抱、适应这种变化。就比如把这变化的一切都写下来，记录下来，分享出来，兴许能给后来者一些启发和帮助。

朋友，不止我这样，你应该也是如此。

如今，这个项目存在的意义，可能就是减少互联网上的死链，还给读者一个仍然可持续访问的博客，未来这个博客将依然存在，但永远进入维护状态，我依然会用脚本把文章推送到这里。

总之，谢谢你能看到这里。
